# Feasibility Review

Model feasibility as constraints, not ambition.

## Workflow

1. List timeline, compute, data access, implementation, evaluation, writing, ethics, and publication risks.
2. Separate feasible MVP from optional extension.
3. Penalize single-person 12-week plans that require unavailable data, GPU scale, or production deployments.
4. Write `docs/diagnosis/feasibility_report.md`.

## Guardrails

- Do not propose experiments requiring unavailable resources.
- State assumptions and blockers concretely.
- Tie feasibility risks back to the revised experiment plan.
